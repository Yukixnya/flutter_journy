# Flutter Journy

`Flutter Journy` is a collection of small Flutter practice projects and assignments. Each folder is a standalone app with its own `pubspec.yaml`, platform folders, and app-specific README.

## Project Catalog

| Folder | Notes |
| --- | --- |
| `assingment2.1/` | Dart exercise project. |
| `assingment3.2/` | Dart exercise project. |
| `assingment4/` | Multi-screen help/instruction app with navigation, saved items, and SnackBars. |
| `assingment5/` | Flutter assignment project. |
| `assingment6/` | Flutter assignment project. |
| `demo/` | Starter demo app. |
| `dice_app/` | Dice app exercise. |
| `journal_app/` | Journal entry app with saved entries. |
| `quote_gen_app/` | Quote generator app. |
| `recipe_app/` | Recipe/help instruction app with saved items and scrollable screens. |
|`fitness_app/`| Animation filled application using fitness as app as theme|

## How To Run An App

1. Open the folder for the app you want to run.
2. Install dependencies with `flutter pub get`.
3. Start the app with `flutter run`.

Example:

```bash
cd recipe_app
flutter pub get
flutter run
```

## Useful Commands

- `flutter analyze` to check for analyzer issues.
- `flutter test` to run widget tests.
- `flutter clean` if a build gets stuck after dependency or platform changes.

Each project folder contains its own README with screenshots and feature notes. Use those files for app-level documentation, and keep this file as the repo landing page.
